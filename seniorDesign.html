Senior Design Project

Collecting Feedlot Closeout Data with ISU Feedlot Monitor System

Client Company: Iowa Beef Center (ISU Extension)

Client: Garland Dahlke

Faculty Advisor: Dr. Ali Jannesari

View Project Website

Problem Statement

Iowa State uses feedlot closeout data for research, supported by a desktop application that farmers across the United States use to submit their data. Currently, the application’s data submission process does not work, and the software itself is outdated with limited support. Our goal is two-fold: first, to restore reliable data transfer functionality, and second, to modernize the application to ensure stability and long-term support. This is crucial for Iowa’s farming community—by fixing the feedlot data process, farmers will gain timely insights needed to thrive, strengthening the state’s agricultural backbone.

Intended Users

The intended users of this project are farmers across the continental United States, along with ISU researchers and partner institutions. Iowa Beef has long provided quarterly reports with valuable feed insights to farmers statewide. Due to blocked networks and outdated systems, this process is currently halted. Restoring and improving the application will once again provide clear, consistent, and dependable reports—helping farmers succeed and ensuring researchers can collect and analyze essential data.

Technical Approach

The existing VB6 codebase will be migrated to modern languages such as VBA, Java, and Python for maintainability and longevity. Additionally, we are implementing a secure FTP system to allow client data submissions without being blocked by university network policies. These improvements will modernize the system and ensure that the data pipeline remains reliable well into the future.
